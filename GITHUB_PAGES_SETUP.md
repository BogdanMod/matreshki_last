# Настройка GitHub Pages

## Шаги для активации GitHub Pages:

1. **Откройте репозиторий на GitHub:**
   - Перейдите на https://github.com/BogdanMod/matreshki_last

2. **Откройте настройки репозитория:**
   - Нажмите на вкладку **Settings** (в верхнем меню репозитория)

3. **Найдите раздел Pages:**
   - В левом меню найдите и нажмите на **Pages**

4. **Настройте источник:**
   - В разделе **Source** выберите **GitHub Actions**
   - НЕ выбирайте "Deploy from a branch"

5. **ВАЖНО - НЕ добавляйте кастомный домен:**
   - НЕ заполняйте поле "Custom domain"
   - Оставьте его пустым
   - GitHub Pages автоматически создаст URL: `https://BogdanMod.github.io/matreshki_last/`

6. **Сохраните:**
   - GitHub Actions автоматически начнет деплой после первого пуша

## После настройки:

- GitHub Actions workflow автоматически соберет и задеплоит проект
- Сайт будет доступен по адресу: **https://BogdanMod.github.io/matreshki_last/**

## Проверка деплоя:

1. Перейдите на вкладку **Actions** в репозитории
2. Убедитесь, что workflow "Deploy to GitHub Pages" выполнился успешно
3. После успешного деплоя (обычно 2-3 минуты) сайт будет доступен

## Если видите ошибку о кастомном домене:

- **НЕ добавляйте кастомный домен** в настройках Pages
- Оставьте поле "Custom domain" пустым
- GitHub Pages автоматически создаст правильный URL
- Если вы уже добавили кастомный домен, удалите его из настроек

## Если что-то не работает:

- Проверьте, что в настройках Pages выбран источник **GitHub Actions** (не branch)
- Проверьте, что поле "Custom domain" пустое
- Проверьте логи в разделе **Actions** на наличие ошибок
- Убедитесь, что workflow файл `.github/workflows/deploy.yml` находится в репозитории
